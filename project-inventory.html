Scalable Action-RPG Inventory Plugin for Unreal Engine · 2026

A C++ / UMG inventory plugin for Unreal Engine designed for action-adventure and RPG projects. The system is built to be game-ready, scalable, plugin-based, and easy to migrate into other projects, with support for item categories, stackable consumables, permanent fixed-slot items, limited reusable items, popup actions, item descriptions, and Gameplay Ability System integration.


The UI and interaction style are inspired by Lies of P, with a main inventory menu, category-based inventory pages, selectable slots, contextual item popups, and detailed item panels. The system is currently work in progress, but already supports core gameplay inventory behaviors such as starting items, permanent limited-use items, stack consumption, item selection, popup actions, and inventory page switching.

  • Designed the plugin architecture around reusable Unreal Engine C++ and UMG classes
  • Implemented the main inventory menu root with page switching between main menu, equipment, and spatial inventory pages
  • Built category-based inventory grids for Basic Items, Equippables, Consumables, and Craftables
  • Created a replicated inventory list using Fast Array-style inventory entries and runtime inventory item objects
  • Implemented item manifests using InstancedStruct based fragments for modular item data and behavior
  • Built support for stackable consumables, permanent fixed-slot items, and limited or unlimited reusable items
  • Implemented server-side consume logic for both limited reusable items and normal stackable items
  • Built item description logic using the Composite Design Pattern (Composite / Leaf) widget architecture driven by item fragments and gameplay tags
  • Structured the plugin to be portable and easy to migrate into other Unreal Engine projects
  • Uses a Menu Root Widget as the top-level inventory UI controller, responsible for switching between main menu, equipment page, and spatial inventory page
  • Uses the InventoryComponent class to own replicated inventory data, create the inventory menu widget, handle pickups, add starting items, consume items, and open or close the inventory UI
  • Uses two Structures, InventoryFastArray and InventoryEntry to replicate runtime Inventory Item objects as inventory entries
  • Uses Item Manifest and InstancedStruct fragments to define item behavior in a modular way instead of hardcoding behavior by category
  • Separates item category from item behavior: category controls where an item appears, while fragments control what the item does
  • The plugin is designed to integrate with external systems such as stats, consumption effects, equipment logic, and Gameplay Effects/Abilities
